# Excel 导入说明 (EXCEL_IMPORT)

本文档详细说明本项目期望的 Excel 文件格式, 以及如何让你的数据被正确识别。

> 💡 **不确定格式时, 请先下载 [`public/data/中国公司作息情况.example.xlsx`](../public/data/中国公司作息情况.example.xlsx) 作为模板。**
>
> 当前版本仅支持 `.xlsx` / `.xls` 格式, 暂不支持 CSV 上传。如果你手头只有 CSV, 请先用 Excel/WPS 另存为 `.xlsx` 再上传。

## V2 公司点位字段 (可选)

V2 新增 6 个可选字段 (列 I-N), 用于支持**公司点位地图**:

| 列 | 字段 | 示例 | 说明 |
| -- | ---- | ---- | ---- |
| I  | 省份 | 广东省 | 缺省时从城市中心库推断 |
| J  | 区县 | 海淀区 | |
| K  | 详细地址 | 中关村大街1号 | |
| L  | 经度 | 116.404 | 中国范围 73-135 东经 |
| M  | 纬度 | 39.915 | 中国范围 18-54 北纬 |
| N  | 坐标系 | gcj02 | wgs84 / gcj02 / bd09, 缺省=unknown |

填写规则:
- 有合法经纬度 → 记录在"公司点位模式"显示为精确点位
- 无经纬度 → 退回城市中心点位 (geo_level = 'city')
- 经纬度必须在中国范围内 (东经 73°-135°, 北纬 18°-54°) 才被视为有效

## V2.5 扩展字段 (可选, 列 O-P)

V2.5 新增 2 个可选字段, 用于手动标注地理精度和来源:

| 列 | 字段 | 示例 | 说明 |
| -- | ---- | ---- | ---- |
| O  | 地理精度 | coordinate | coordinate/address/district/city/unknown, 缺省时按经纬度有无推断 |
| P  | 地理来源 | excel | manual/excel/api/geocoded/unknown, 缺省时按坐标来源推断 |

## V2.5 标准化明细表格式 (可选)

除了三段式格式 (955/965/996 区域), V2.5 还支持**标准化明细表格式**: 一行一条作息记录, 表头自定义列名。

表头列 (中英文均可, 大小写不敏感):

| 列名别名 | 说明 |
| -------- | ---- |
| 公司 / company | 公司名称 (必填) |
| 城市 / city | 城市名 |
| 省份 / province | 省份 |
| 区县 / district | 区县 |
| 详细地址 / address | 详细地址 |
| 经度 / lng / lon | 经度 |
| 纬度 / lat | 纬度 |
| 坐标系 / coord_system | wgs84/gcj02/bd09 |
| 地理精度 / geo_level | coordinate/address/district/city/unknown |
| 地理来源 / geo_source | manual/excel/api/geocoded |
| 工作制度 / work_system | 955/965/996/997/007/大小周/单休/排班/加班 |
| 周末类型 / weekend_type | 双休/单休/大小周/排班/轮休 |
| 时间 / time | 时间或日期 |
| 规则 / rule | 规则描述 |
| 证据 / evidence | 证据 (多个用 \| 或 ; 分隔) |
| 区域 / section | 955/965/996 |
| 可信度 / confidence | A/B/C/D/E |

解析器自动识别格式: 如果第一行是表头且含"公司"列 + 至少 3 个已知字段, 则按明细表解析; 否则按三段式解析。

### 招聘站 / 多来源扩展列 (可选)

明细表额外支持以下招聘站字段 (来自 `scripts/ingest-recruit.ts` 的产出, 见 [PRD-0002](requirements/PRD-0002-recruit-ingestion.md)):

| 列名别名 | 说明 |
| -------- | ---- |
| 部门 / department | 部门, 如 "研发中心" |
| 岗位 / 职位 / job_title / position | 岗位/职位名, 如 "后端工程师" |
| 上班时间 / 上班 / work_begin | 上班时间, 如 "09:00" |
| 下班时间 / 下班 / work_end | 下班时间, 如 "21:00" |
| 一周工作天数 / 工作天数 / workdays | 每周工作天数, 如 "5" / "5.5" / "6" |
| 来源平台 / 平台 / source_platform | 来源平台, 如 "BOSS直聘" / "51job" / "智联招聘" |
| 来源链接 / 链接 / source_url | 职位页 URL (作为证据, 详情页可点击"查看来源页") |
| 采集时间 / collected_at | 采集日期, 如 "2026-06-27" |

> `来源链接` 会经 URL 安全校验 (仅 http/https), 非法链接被丢弃。这些字段为空时不写入, 不影响老数据。

## 1. 工作表 (Sheet) 要求

- 至少一个工作表, 名称**任意** (项目通过内容识别, 不依赖 sheet 名)
- 推荐单个 sheet 内含三个区域 (见下方)
- 也支持多 sheet, 每个 sheet 内部按相同规则解析

## 2. 三个区域的结构

每个区域由**一行区域标题** + 可选空行 + **一行字段表头** + **数据行**组成。

### 2.1 区域标题 (会触发 section 切换)

| 标题原文                                       | 识别为 section |
| ---------------------------------------------- | -------------- |
| 包含 `955` 和 `正常` (e.g. "955正常工作制度公司名单") | `955`          |
| 包含 `965` 和 `较差` (e.g. "965较差工作制度公司名单") | `965`          |
| 包含 `996` 和 (`高强度记录` 或 `高强度`) (e.g. "996...公司高强度记录") | `996`          |

### 2.2 字段表头 (区域标题后的第一个非空行)

表头行第一列若为 `城市`, 则视为表头, 跳过。

### 2.3 数据行 (表头之后到下一个区域标题之前的所有非空行)

#### 955 区域

仅 2 列:

| 列 | 字段        | 是否必填 |
| -- | ----------- | -------- |
| A  | 城市        | ✅        |
| B  | 公司        | ✅        |

#### 965 / 996 区域

5-8 列 (基础字段) + 6 列可选 (V2 公司点位):

| 列 | 字段     | 是否必填 | 说明 |
| -- | -------- | -------- | ---- |
| A  | 城市     | ✅        | 支持多城市分隔符 |
| B  | 公司     | ✅        | |
| C  | 时间     | ❌        | 可为 Excel 数字日期或文本 |
| D  | 规则     | ❌        | |
| E  | 证据 1   | ❌        | |
| F  | 证据 2   | ❌        | 996 区域常见 |
| G  | 证据 3   | ❌        | |
| H  | 证据 4   | ❌        | |
| I  | 省份     | ❌ (V2)   | e.g. "广东省", 缺省时从城市中心库推断 |
| J  | 区县     | ❌ (V2)   | e.g. "海淀区" |
| K  | 详细地址 | ❌ (V2)   | e.g. "中关村大街1号" |
| L  | 经度     | ❌ (V2)   | e.g. 116.404, 中国范围 73-135 |
| M  | 纬度     | ❌ (V2)   | e.g. 39.915, 中国范围 18-54 |
| N  | 坐标系   | ❌ (V2)   | wgs84 / gcj02 / bd09 (火星/百度/国测局), 缺省=unknown |

> 多个证据列会被自动合并为 `evidence_list` 数组和 `evidence_text` 字符串 (用 ` | ` 分隔)。

**V2 公司点位逻辑**:
- 如果记录有合法经纬度 (列 L/M 在中国范围内), 用它作为公司精确坐标, `geo_level = 'coordinate'`
- 如果没有经纬度, 退回城市中心坐标, `geo_level = 'city'`
- 有公司经纬度的记录不按多城市展开 (经纬度是单一位置)
- 公司点位模式下, 每条有坐标的记录在地图上显示为独立点位

## 3. 字段格式细则

### 3.1 城市 (`城市`)

**支持多城市**, 用以下任意分隔符:

| 分隔符 | 示例                  |
| ------ | --------------------- |
| `/`    | `北京/上海`           |
| `,`    | `北京, 西安`          |
| `，`   | `北京，西安` (中文逗号) |
| `-`    | `深圳-成都-重庆`      |
| `、`   | `北京、上海`          |
| 空格   | `北京 上海`           |

**多城市会自动展开为多条记录** (e.g. `北京/上海 - Apple` → `Apple-北京` + `Apple-上海`)。

**省名自动转换** (e.g. `广东` → 广州坐标, `山东` → 济南坐标, `广西` → 南宁坐标)。

**非城市关键词会被过滤** (e.g. `remote`, `Remote`, `远程`, `不限`, `全国`, `多地`)。

**城市名别名容错** (e.g. `深圳市` → `深圳`, `合肥市` → `合肥`, `马鞍山市` → `马鞍山`, `云南昆明` → `昆明`)。

### 3.2 公司 (`公司`)

- 自动去除前后空格
- 保留原始值到 `company_name_raw`
- 中英文括号、别名**不拆分** (e.g. `Udesk(北京沃丰时代数据科技有限公司)` 保留为整体)
- 若城市列为空但公司名含 ` - ` (e.g. `Linkedin - 北京`), 自动从公司名提取城市

### 3.3 时间 (`时间`)

支持两种格式:

| 输入        | 解析为                       |
| ----------- | ---------------------------- |
| Excel 数字 (e.g. `43556`) | `2019-01-01` (Excel epoch 1900-01-01 = 1) |
| ISO 字符串 (e.g. `2024-03-15`) | `2024-03-15`                |
| 其他文本    | 保留原值                     |

### 3.4 规则 (`规则`)

自由文本, 用于 `work_system` / `weekend_type` / `risk_level` 的关键词识别。

示例:
- `9:00-18:00-5;上下班弹性半小时;午休1h;` → work_system=965, weekend_type=双休, risk=medium
- `996无偿加班,降薪` → work_system=996, weekend_type=单休, risk=very_high (含"无偿加班"+"降薪")
- `大小周` → work_system=大小周, weekend_type=大小周, risk=high
- `11116、两班倒` → work_system=996 (含"11116"), weekend_type=排班/轮休 (含"两班倒"), risk=very_high

### 3.5 证据 (`证据`)

自由文本, 多列证据会合并。V1 不解析 URL, 仅作为描述性文本展示。

## 5. 常见问题

### Q1: 我的 Excel 用了不同标题, 怎么办?

区域标题只要包含关键字即可, 例如:
- `955 WLB 公司` (含 955 但不含"正常") → ❌ 不会识别
- `955正常公司` (含 955 + "正常") → ✅ 识别为 955 区域

如果你的数据用了完全不同的标题, 可修改 `src/lib/normalize.ts` 中的 `parseSheetRows` 函数中的匹配条件。

### Q2: 我的 Excel 字段顺序不同, 怎么办?

当前 V1 强制字段顺序为 `城市 | 公司 | 时间 | 规则 | 证据`。如需调整, 修改 `src/lib/normalize.ts` 的 `buildCompanyRecord` 函数。

### Q3: 多城市记录如何导出?

在 `normalized_companies.json` 中, 多城市记录会**展开为多条**记录 (同 `source_row`, 不同 `id` 后缀)。
所有展开记录共享相同的 `company_name` / `rule_text` / `evidence_text`, 但 `city` / `lng` / `lat` 不同。

### Q4: 我的公司名含特殊字符, 会被处理吗?

V1 仅做前后空格清理, 不拆分括号、不替换别名。原始值保留在 `company_name_raw`。

### Q5: 上传后的数据会保存吗?

不会。所有解析在浏览器本地完成, 刷新页面后回到预置示例数据。如需保存, 请使用"导出数据"功能下载 JSON / CSV。
