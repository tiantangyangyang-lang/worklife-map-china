# Excel 导入说明 (EXCEL_IMPORT)

本文档详细说明本项目期望的 Excel 文件格式, 以及如何让你的数据被正确识别。

## 1. 工作表 (Sheet) 要求

- 至少一个工作表, 名称**任意** (项目通过内容识别, 不依赖 sheet 名)
- 推荐单个 sheet 内含三个区域 (见下方)
- 也支持多 sheet, 每个 sheet 内部按相同规则解析

## 2. 三个区域的结构

每个区域由**一行区域标题** + 可选空行 + **一行字段表头** + **数据行**组成。

### 2.1 区域标题 (会触发 section 切换)

| 标题原文                                       | 识别为 section |
| ---------------------------------------------- | -------------- |
| 包含 `955` 和 `正常` (e.g. "955正常工作制度公司名单") | `955`          |
| 包含 `965` 和 `较差` (e.g. "965较差工作制度公司名单") | `965`          |
| 包含 `996` 和 (`高强度记录` 或 `高强度`) (e.g. "996...公司高强度记录") | `996`          |

### 2.2 字段表头 (区域标题后的第一个非空行)

表头行第一列若为 `城市`, 则视为表头, 跳过。

### 2.3 数据行 (表头之后到下一个区域标题之前的所有非空行)

#### 955 区域

仅 2 列:

| 列 | 字段        | 是否必填 |
| -- | ----------- | -------- |
| A  | 城市        | ✅        |
| B  | 公司        | ✅        |

#### 965 / 996 区域

5-8 列:

| 列 | 字段     | 是否必填 |
| -- | -------- | -------- |
| A  | 城市     | ✅        |
| B  | 公司     | ✅        |
| C  | 时间     | ❌ (可为 Excel 数字日期或文本) |
| D  | 规则     | ❌        |
| E  | 证据 1   | ❌        |
| F  | 证据 2   | ❌ (996 区域常见) |
| G  | 证据 3   | ❌        |
| H  | 证据 4   | ❌        |

> 多个证据列会被自动合并为 `evidence_list` 数组和 `evidence_text` 字符串 (用 ` | ` 分隔)。

## 3. 字段格式细则

### 3.1 城市 (`城市`)

**支持多城市**, 用以下任意分隔符:

| 分隔符 | 示例                  |
| ------ | --------------------- |
| `/`    | `北京/上海`           |
| `,`    | `北京, 西安`          |
| `，`   | `北京，西安` (中文逗号) |
| `-`    | `深圳-成都-重庆`      |
| `、`   | `北京、上海`          |
| 空格   | `北京 上海`           |

**多城市会自动展开为多条记录** (e.g. `北京/上海 - Apple` → `Apple-北京` + `Apple-上海`)。

**省名自动转换** (e.g. `广东` → 广州坐标, `山东` → 济南坐标, `广西` → 南宁坐标)。

**非城市关键词会被过滤** (e.g. `remote`, `Remote`, `远程`, `不限`, `全国`, `多地`)。

**城市名别名容错** (e.g. `深圳市` → `深圳`, `合肥市` → `合肥`, `马鞍山市` → `马鞍山`, `云南昆明` → `昆明`)。

### 3.2 公司 (`公司`)

- 自动去除前后空格
- 保留原始值到 `company_name_raw`
- 中英文括号、别名**不拆分** (e.g. `Udesk(北京沃丰时代数据科技有限公司)` 保留为整体)
- 若城市列为空但公司名含 ` - ` (e.g. `Linkedin - 北京`), 自动从公司名提取城市

### 3.3 时间 (`时间`)

支持两种格式:

| 输入        | 解析为                       |
| ----------- | ---------------------------- |
| Excel 数字 (e.g. `43556`) | `2019-01-01` (Excel epoch 1900-01-01 = 1) |
| ISO 字符串 (e.g. `2024-03-15`) | `2024-03-15`                |
| 其他文本    | 保留原值                     |

### 3.4 规则 (`规则`)

自由文本, 用于 `work_system` / `weekend_type` / `risk_level` 的关键词识别。

示例:
- `9:00-18:00-5;上下班弹性半小时;午休1h;` → work_system=965, weekend_type=双休, risk=medium
- `996无偿加班,降薪` → work_system=996, weekend_type=单休, risk=very_high (含"无偿加班"+"降薪")
- `大小周` → work_system=大小周, weekend_type=大小周, risk=high
- `11116、两班倒` → work_system=996 (含"11116"), weekend_type=排班/轮休 (含"两班倒"), risk=very_high

### 3.5 证据 (`证据`)

自由文本, 多列证据会合并。V1 不解析 URL, 仅作为描述性文本展示。

## 5. 常见问题

### Q1: 我的 Excel 用了不同标题, 怎么办?

区域标题只要包含关键字即可, 例如:
- `955 WLB 公司` (含 955 但不含"正常") → ❌ 不会识别
- `955正常公司` (含 955 + "正常") → ✅ 识别为 955 区域

如果你的数据用了完全不同的标题, 可修改 `src/lib/normalize.ts` 中的 `parseSheetRows` 函数中的匹配条件。

### Q2: 我的 Excel 字段顺序不同, 怎么办?

当前 V1 强制字段顺序为 `城市 | 公司 | 时间 | 规则 | 证据`。如需调整, 修改 `src/lib/normalize.ts` 的 `buildCompanyRecord` 函数。

### Q3: 多城市记录如何导出?

在 `normalized_companies.json` 中, 多城市记录会**展开为多条**记录 (同 `source_row`, 不同 `id` 后缀)。
所有展开记录共享相同的 `company_name` / `rule_text` / `evidence_text`, 但 `city` / `lng` / `lat` 不同。

### Q4: 我的公司名含特殊字符, 会被处理吗?

V1 仅做前后空格清理, 不拆分括号、不替换别名。原始值保留在 `company_name_raw`。

### Q5: 上传后的数据会保存吗?

不会。所有解析在浏览器本地完成, 刷新页面后回到预置示例数据。如需保存, 请使用"导出数据"功能下载 JSON / CSV。
